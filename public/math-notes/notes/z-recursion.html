What is recursion?

To put simply, recursion is solving a problem referring to itself. It depends on smaller pieces of the same problem in order to solve the main problem. To be more concrete, recursive problems repeatedly calls itself, dividing the main problem into many subproblems of itself. For example, finding the factorial of a number can be done recursively.

Finding the factorial requires it to repeatedly multiply its current value with the next number. The self-referential part of the factorial problem is is the product.

When do I use recursion?

There are a couple of cases where recursion is useful:

  • When you want to find every possible solution of a problem.
  • The object in the problem is defined recursively, like a tree or a graph data structure.
